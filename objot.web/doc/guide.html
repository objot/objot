面向服务的AJAX

Objot是一个轻量级的组件，可以为Web开发项目有效地构建面向服务的AJAX架构。 服务器部署业务数据Model，部署业务逻辑Service；浏览器部署用户界面，部署操作逻辑，并与Service交互。 交互的数据、流程由开发者定义，用Objot实现。

Objot包含了一组Javascript、Java和Python代码，来帮助开发者实现浏览端与服务端的交互。 还包含了一组Javascript代码，来帮助开发者实现用户界面和操作逻辑。 在浏览端，Objot中的函数名均以$开头；在服务端，Objot代码都在objot包中。

一个在线聊天的范例，演示了如何用Objot和面向服务的AJAX架构实现一个完整的Web应用。

定义服务对象Service Object

借助于Objot轻量级对象序列化反序列化机制，浏览端和服务端之间的数据交互简洁而有效。 可以在交互中传递的数据类型包括：

JavascriptJavaPython
null undefinednullNone
booleanbooleanbool
numberint long(部分) float double Integer Long(部分) Float Doubleint long(部分) float
stringStringunicode str(UTF-8)
Arrayboolean[] int[] long[] <T>[] List<?>list tuple
Object(非Function String Number Boolean Array)Object Map<String,Object>object(非instance) dict
对象之间可以任意引用，整个对象图都可以被传递，同时也可以定义规则，只传递对象图的局部（子图）。 Objot支持单继承类，服务端和浏览端可以使用不同的类名，可以使用不同的属性名，均可以对应。 传递的对象图必须以list或者object为起点，传递的对象被称为服务对象Service Object。

浏览端使用Service Object需要先定义相应的类。
