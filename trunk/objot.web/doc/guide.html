面向服务的AJAX

Objot是一个轻量级的组件，可以为Web开发项目有效地构建面向服务的AJAX架构。 服务器部署业务数据Model，部署业务逻辑Service；浏览器部署用户界面，部署操作逻辑，并与Service交互。 交互的数据、流程由开发者定义，用Objot实现。

Objot包含了一组Javascript、Java和Python代码，来帮助开发者实现浏览端与服务端的交互。 还包含了一组Javascript代码，来帮助开发者实现用户界面和操作逻辑。 在浏览端，Objot中的函数名均以$开头；在服务端，Objot代码都在objot包中。 浏览端，Objot兼容Firefox 2.0，Firefox 1.5，IE 7（测试中），IE 6；服务端，Objot要求Java 5(1.5)以上，或者Python 2.4以上。

一个在线聊天的范例，演示了如何用Objot和面向服务的AJAX架构实现一个完整的Web应用。

服务对象Service Object

借助于Objot轻量级对象编码解码机制，浏览端和服务端之间的数据交互简洁而有效。 可以在交互中传递的数据类型包括：

JavascriptJavaPython
null undefinednullNone
booleanbooleanbool
numberint long(部分) float double Integer Long(部分) Float Doubleint long(部分) float
stringStringunicode str(UTF-8)
Arrayboolean[] int[] long[] <T>[] List<?>list tuple
Object(非Function String Number Boolean Array)Object Map<String,Object>object(非instance) dict
其中，字符串是Unicode格式（Python服务端可以配置为UTF-8，但不推荐），并且不能包含一个特殊字符：'\20'（即\u0010）。 Objot将数据编码成一个字符串，并使用了特殊字符'\20'。

list、object之间可以任意引用，构成的整个数据图都可以被编解码；同时也可以定义规则，只处理数据图的局部（子图）。 Objot支持单继承类，服务端和浏览端可以使用不同的类名，可以使用不同的属性名，均可以对应。 传递的数据图必须以list或者object为起点，传递的对象被称为服务对象Service Object（下称SO）。

在浏览端使用SO

浏览端使用Service Object需要先定义相应的类。例如定义Ok和Err表示服务成功和失败： 

Ok = function (hint) {
	this.hint = $(hint);
}
Err = function (hint) {
	this.hint = $(hint);
}
ErrUnsigned = function (hint) {
	this.hint = $(hint);
}
$class('Ok'), $class('Err');
$class('ErrUnsigned', Err); // ErrUnsigned的父类是Err
3个function定义了构造函数，$class把这些function定义为类。 使用$get可以将数据编码为字符串： 
x = new Ok("I'm Ok");
y = $get(x, Object);
$get可以理解为get from object，得到的字符串可以再被解码为数据： 
x2 = $set(y);
x2.hint == x.hint || $throw('error'); // 如果hint不同，就抛出异常
$set可以理解为set to object。
